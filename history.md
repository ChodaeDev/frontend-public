# 작업 일지

## 2026-06-02

- 상담게시판 목록/상세 API에 로그인 사용자 기준 `isOwner` 값을 반영했다. 목록/검색 응답의 소유자 판정과 상세 응답 필드를 정리하고, 프론트엔드 상세 화면도 `post.isOwner`를 사용하도록 수정했다.
- `users.level`이 `admin` 또는 `superadmin`인 계정은 상담게시판의 모든 글과 댓글을 조회/수정/삭제할 수 있도록 권한 판정을 확장했다. 관리자 계정은 상담 목록에서 작성자 이름 마스킹을 적용하지 않도록 변경했다.
- 상담게시판 작성자 이름 마스킹을 첫 글자만 노출하고 나머지 글자 수만큼 `*`를 표시하도록 수정했다. 예: `한상대` -> `한**`, `flower` -> `f*****`.
- 로그인 응답 생성 시 DB에서 반환된 사용자 `id`가 `Long`인 경우에도 처리되도록 `Number.intValue()` 변환을 적용해 `Long cannot be cast to Integer` 오류를 수정했다.
- 댓글 계층 구조를 지원하도록 `child_comment` 기능을 추가했다. 댓글 생성/조회 DTO, 공통 댓글 Mapper, 각 게시판별 Comment Mapper XML, 서비스 계층을 수정해 대댓글 데이터를 API 응답에 포함하도록 정리했다. (`89e9241`)
- 댓글 테이블에 부모 댓글 참조용 `parent_comment_id` 컬럼을 추가해 일반 댓글과 대댓글을 같은 댓글 구조 안에서 연결할 수 있도록 DB 스키마를 변경했다. (`89e9241`)
- 상담게시판 상세 화면에서 댓글/대댓글 표시와 작성 흐름을 반영하도록 NextJS 컴포넌트와 타입을 갱신했다. (`89e9241`)
- 게시글/댓글 스키마의 공통 필드와 삭제 상태 처리(`is_deleted`)를 디버깅했다. 게시글/댓글 Entity, DTO, Repository, Mapper, Service 전반을 수정해 통합 스키마 기준으로 API 응답 구조를 맞췄다. (`5dedf79`)
- `chodae_recovery.sql`의 게시글/댓글 관련 DB 스키마를 최신 백엔드 구조에 맞게 반영했다. (`89e9241`, `5dedf79`)
